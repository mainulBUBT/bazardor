# Fix: Remove nested container-fluid from edit.blade.php

## Problem
The `show.blade.php` file does NOT have `<div class="container-fluid">` wrapper because it was already fixed. However, `edit.blade.php` still has an extra `<div class="container-fluid">` wrap (line 5) that creates double-nesting, since `layouts/admin/app.blade.php` already provides one at line 88.

## Root Cause
- `layouts/admin/app.blade.php` line 88: `<div class="container-fluid">@yield('content')</div>`
- `admin/users/edit.blade.php` line 5: adds another `<div class="container-fluid">` — causes nested containers

## Analysis of user files
| File | status |
|------|--------|
| `show.blade.php` | No wrapper (already fixed) |
| `edit.blade.php` | Has wrapper (needs fix) |
| `create.blade.php` | No wrapper (correct) |
| `index.blade.php` | No wrapper (correct) |

## Fix
Remove from `resources/views/admin/users/edit.blade.php`:
1. Line 5: `<div class="container-fluid">`
2. Line 205: `</div>` (closing tag for the container)

This will make `edit.blade.php` consistent with `show.blade.php`, `create.blade.php`, and `index.blade.php`.
