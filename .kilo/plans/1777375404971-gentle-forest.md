# Fix: Product Creation Fails Due to Missing Slug

## Problem
The `POST /api/products/create` endpoint fails with:
```
SQLSTATE[HY000]: General error: 1364 Field 'slug' doesn't have a default value
```

The `products` table has a `slug` column defined as `string('slug')->unique()` with no default value, but `ProductService::store()` does not generate or include a slug when creating products. This also affects the admin product creation flow since the admin form has no slug input field.

## Root Cause
`app/Services/ProductService.php:71-88` - The `create()` call does not include a `slug` field:
```php
$product = $this->product->create([
    'name' => $data['name'],
    'category_id' => $data['category_id'],
    // ... no 'slug' => ...
]);
```

The admin form (`resources/views/admin/products/create.blade.php`) has no slug input field, meaning slug is never provided by any user flow.

## Solution
Add automatic slug generation in `ProductService::store()` using Laravel's `Str::slug()`. Always append a UUID suffix to the name-based slug to guarantee uniqueness without collision checks.

### Files to Modify

**app/Services/ProductService.php**

1. Add `use Illuminate\Support\Str;` to imports (line 8, after existing imports)

2. In the `store()` method, add slug generation logic before the `create()` call (after line 69 `unset($data['image']);`):

```php
// Generate unique slug: name + uuid
$slug = Str::slug($data['name']) . '-' . Str::uuid();
```

3. Add `'slug' => $slug,` to the `create()` array (line 72, right after `'name'`).

### Updated store() method (lines 62-88):
```php
public function store(array $data): Product
{
    return DB::transaction(function () use ($data) {
        // Handle image upload if present
        if (isset($data['image']) && $data['image']->isValid()) {
            $data['image_path'] = handle_file_upload('products/', $data['image']->getClientOriginalExtension(), $data['image']);
        }
        unset($data['image']);

        // Generate unique slug: name + uuid
        $slug = Str::slug($data['name']) . '-' . Str::uuid();

        $product = $this->product->create([
            'name' => $data['name'],
            'slug' => $slug,
            'category_id' => $data['category_id'],
            'unit_id' => $data['unit_id'],
            'description' => $data['description'] ?? null,
            'status' => $data['status'] ?? 'active',
            'is_visible' => $data['is_visible'] ?? true,
            'is_featured' => $data['is_featured'] ?? false,
            'image_path' => $data['image_path'] ?? null,
            'sku' => $data['sku'] ?? null,
            'barcode' => $data['barcode'] ?? null,
            'brand' => $data['brand'] ?? null,
            'base_price' => $data['base_price'] ?? 0,
            'country_of_origin' => $data['country_of_origin'] ?? null,
            'added_by' => $data['added_by'] ?? 'admin',
            'added_by_id' => $data['added_by_id'] ?? null,
            'device_id' => $data['device_id'] ?? null,
        ]);
```

Also verify admin product creation still works via the web form.

## Additional: Update Method
The `update()` method (lines 146-160) also doesn't handle slug. If the product name is updated, regenerate the slug with UUID:

```php
// Regenerate slug if name changed
if (isset($data['name']) && $data['name'] !== $product->name) {
    $slug = Str::slug($data['name']) . '-' . Str::uuid();
} else {
    $slug = $product->slug; // keep existing
}

$product->update([
    'name' => $data['name'] ?? $product->name,
    'slug' => $slug,
    // ... rest of fields
]);
```
